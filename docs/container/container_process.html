コンテナのプロセス

Linuxにおける一般的なコンテナは，幾つかの機能を組み合わせることにより実現されている．その中でも代表的なLinuxのカーネル機能について説明する．

Namespace

Namespaceは，Linuxカーネルのリソースを各プロセスごとに分離するための機構である．この機能により，プロセスは他のプロセスと隔離されるため，互いに干渉せずに環境構築が可能となる．

Namespaceの種類

  • リンゴ
  • イチゴ
  • みかん

画像

Namespace AとBがあり，その中では独立したファイル構造やプロセス空間を持つことができる．プロセス空間については，ホストOS上で動作しているPIDとコンテナ上で動作しているPIDをマッピングしているテーブルがあり，ホストOS上ではPIDが2だが，コンテナ上ではPIDが1である，ということになる．


cgroup

cgroupは，プロセスをグループ化し，Linuxカーネルのリソースの使用を制限する役割を担う．上記のNamespaceはホスト名やPIDなどのリソースを制御するのに対し，cgroupはCPUやメモリといった物理的なリソースを制御する．cgroupを使うことで，特定のプロセスがホストOSの持つ有限なリソースを使い尽くして，ホストOSや他のプロセスに影響を与えないようにコントロールすることができる．


Capability

Capabilityとは，root権限を細分化し，プロセスに特権全てを付与するのではなく，必要最低限の権限を与える機能である，もし，root権限がそのまま付与されたプロセスに脆弱性があった場合に，不正な操作により，システム全体を第三者に乗っ取られる危険性がある．その為，Capabilityを用いてプロセスに与える権限を最小限に制御することで，プロセスに脆弱性があったとしても，ホストOSや他のプロセスへの影響範囲を狭めることができる．
