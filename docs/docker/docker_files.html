Dockerの構成要素

Docker Image

Docker Imageとは，Dockerコンテナを起動するために，必要な設定ファイルをまとめたテンプレートのことである．”Imageはコンテナの元であり，Imageからコンテナを起動する”と理解すると良い．本来はソフトウェアをインストールする際に，様々な設定をカスタマイズをする必要があるが，Docker Imageを使うことで，その設定作業を省略することができる．また，DockerではこのDocker Imageを共有することで，様々なマシンで同じコンテナ(実行環境)を動作させることができる．なお，多くの場合，Docker Imageは他のDocker Imageをベースにして，さらにカスタマイズを加えたものである．

Tag

Docker Imageには，"Tag"という概念がある．これは"Docker Imageのversion"を表している．Docker Imageにタグを付与することで，Docker Imageを区別することができ，Docker Image毎にバージョンで管理したい場合は，タグにバージョン番号を指定する．Docker Imageを共有できるDocker Hubでは，Docker Imageに様々なタグが存在し，どのversionを使用するかを選択できる．Docker Hubからダウンロードする際に，タグ名を何も指定しないと，自動的に"latest"タグが使用されるようになっている．

Docker Imageの構造

Docker Imageは以下のような特徴を持つ．
  1. layer構造になっている
  2. 一度作成されたimageは編集不可能（読み取り専用）
  3. 移動・共有が容易

Docker Imageは，レイヤー構造であり，複数のファイルやディレクトリなどがレイヤ（層）として積み重なり，1つに統合されたファイルシステムとして見えるようになっている．1つのインストールコマンドで，1つのレイヤーが積み重なっていくイメージである．Docker内部のストレージ・ドライバが，レイヤーを積み重ねる役割を担っている．
また、土台となるベースレイヤーの上に，複数のレイヤーを積み重ねた編集不可能(Read Only)のイメージレイヤーが纏められており，コンテナ起動時に薄い読み書き可能なレイヤー(Thin R/W layer)が，自動的に割り当てられる．Docker Imageに対する変更情報（ファイルの追加・削除など）は，すべてこのコンテナ用レイヤー上に記録され，このレイヤーをコミットすることで，新しいDocker Imageを生成することができる．

また，Docker Imageは移動や共有がし易い，という特徴がある．同一ホスト上で動く複数のコンテナがある場合は，イメージレイヤーを共有することができ，ホストのディスク容量を抑制することができる．Docker Imageは，各レイヤーをファイルやコードの差分のように扱えるため，ネットワークを通じて共有する際に，トラフィックや時間の節約も可能となる．

Dockerfile

Dockerfileは，Dockerで作成するDocker Imageを管理するためのファイルである．言わば，Docker Imageの設計図である．Dockerfileに，Docker Imageを作成する際の定義（どのようなファイルを追加するか，どんなコマンドを実行するか）を記述する．Dockerfileの各命令は，Docker Imageのレイヤを作成する．Dockerfileを使用してDocker Imageを作成することをビルドと呼び，作成されたDocker Imageを起動することで，実際にコンテナが立ち上がる．Dockerfileを変更してDocker Imageをリビルドすると，変更されたレイヤだけがリビルドされる．これが，他の仮想化技術と比較して，Docker Imageを非常に軽量・小型・高速にする要因の一つであると言える．

Dockerfileのメリット

  • コンテナ構築の一連の手順を自動化できる
  • Dockerコンテナの仕様をファイルで管理できる
  • 自分の好きなようにカスタマイズすることができる
  • 共有することで，複数人が簡単に同じ環境を構築できる
  • Docker Imageに、変更内容を履歴として保持することができる
基本的にDocker Hubなどからpullしてきた既存のDocker Imageにファイルを追加したり，コマンドを実行したりすることで作成する． 例として，app.pyというPythonプログラムを実行するためのコンテナのDockerfileは，以下のようになる．（※Linuxコンテナ・Docker入門(後編)より引用） 
                    
                        # Pythonの公式イメージから作成
                        FROM python:3.7.6-stretch

                        # pipでパッケージをインストール
                        RUN pip install pip --upgrade && \
                            pip install numpy scipy

                        # aptでパッケージをインストール
                        RUN apt-get update -y && \
                            DEBIAN_FRONTEND=noninteractive apt-get install -y --no-install-recommends \
                            sshpass

                        # ディレクトリを作成
                        WORKDIR /work

                        # プログラムを配置
                        COPY app.py /work

                        # コンテナ起動時に最初に実行するコマンドを指定
                        ENTRYPOINT python /work/app.py

Dockerfileのコマンド例

  • FROM：基となるDocker Image
  • LABEL：ラベル情報（メタデータ）
  • COPY：ホストからコンテナへのファイルコピー
  • RUN：docker build 時に実行するコマンド
  • CMD：docker run 時に実行するコマンド（コンテナのデフォルトの実行コマンド）
  • ENTRYPOINT：コンテナのエントリーポイントの定義
  • EXPOSE：コンテナで利用するポート番号の定義

これらの他にも様々なコマンドがある．詳細はDockerfileの公式リファレンスを参照する．

Dockerの主要コマンド

Dockerコンテナを操作する際に，よく使われるコマンドを紹介する．

  • $ docker run <イメージ名>：イメージの構築，コンテナの構築・起動
  • $ docker start <コンテナID> or <コンテナ名>：コンテナの開始
  • $ docker stop <コンテナID> or <コンテナ名>：コンテナの停止
  • $ docker restart <コンテナID> or <コンテナ名>：コンテナの再起動
  • $ docker exec <コンテナID> <コマンド>：起動中のコンテナでコマンド実行，コンテナ内にログイン
  • $ docker ps -a：コンテナの一覧の確認
  • $ docker images：イメージの確認
  • $ docker rm <コンテナID> or <コンテナ名>：停止中のコンテナの削除
  • $ docker rmi <イメージID>：イメージの削除
  • $ docker build ：Dockerfileからイメージを作成
  • $ docker logs <コンテナID>：コンテナのログ出力
  • $ docker image pull <イメージ名>:：Docker Hubからイメージをダウンロードする
