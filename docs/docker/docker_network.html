Dockerのネットワーク

Namespaceの分離とコンテナ間通信

通常のLinuxでは，複数のプロセスがポートを重複して開くことはできない．例えば，ApacheとNginxは，デフォルトでTCPポート80番を使用するが，それらを同時に使用することはできない．

一方で，複数のプロセスをコンテナとして実行すれば，PIDといった名前空間が隔離されるため，互いに影響を与えず動作することができる．また，コンテナごとにネットワークも隔離されるため，コンテナ内で各々のプロセスがポートを開放していても，（ホスト側で使用するポート番号が重複していなければ）互いに影響を与えずに起動し続けられる．

これは，Docker Composeを使う場合も同じである．Docker Composeでアプリケーションを実行すると，そのプロセスの名前空間やネットワーク，ボリュームなども，アプリケーションごとに分離することができる．

Composeを用いると，YAMLファイルに記述した，同じComposeのプロジェクト内に存在するコンテナ間で通信可能なネットワークが作成される．これは，User Defined Bridge Network（ユーザ定義ブリッジ・ネットワーク）と呼ばれ，コンテナ間通信を行うだけでなく，インターネットといったホスト上にあるネットワークとの通信もやりとりする．

Dockerのネットワークモデル

通常，Dockerコンテナは，デフォルトではコンテナ外部のネットワークと通信することはできない．そのため，単に複数のコンテナを実行しても，相互に通信できない場合がある．意図した通りにコンテナ間で通信を行うには、適切な設定が必要でである．

$ docker runで，コンテナを起動すると，通常は"bridge0"という名前のブリッジ・ネットワークに接続する．これがデフォルトのブリッジ・ネットワーク(bridge network)である．同じブリッジ・ネットワーク上に接続しているコンテナは，互いに通信が可能となる．そのため，ネットワークを特に定義しないコンテナ間では，コンテナが持つIPアドレスを用いて互いに通信できる．

しかし，この場合，コンテナのIPアドレスが変わった場合に対処が必要となる．コンテナ技術により，容易にアプリケーションをスケールアウトし易くなったが，コンテナ間の通信でIPアドレスが必要となれば，通常のサーバ運用と同じ，あるいはそれ以上に設定・管理が煩雑になる．

Dockerには，コンテナ間での通信や，処理の負荷分散を行うため，ネットワーク内部で利用可能な”コンテナ名”，”IPアドレス”の名前解決（サービス・ディスカバリ）を行う仕組みが，標準で搭載されている．つまり，コンテナ間の通信は，IPアドレスではなく，コンテナ名を用いることができる．これにより，スケールできる環境においてもIPアドレスを意識する必要がなくなり，コンテナ間の通信や負荷分散が容易となる．

しかし，注意点としてデフォルトの”bridge0”という名前のブリッジ・ネットワークでは，上記の名前解決機能が搭載されていない．コンテナ間の通信で名前解決を有効化するには，ブリッジ・ネットワークを作成する必要がある．これをUser Defined Bridge Network（ユーザ定義ブリッジ・ネットワーク）と呼ぶ。なお，$ docker network lsで，Dockerのネットワークを確認する．

Docker Network

Dockerには，デフォルトで存在する3つのネットワークと独自で定義するネットワークがある．

Bridgeネットワーク

Brightネットワークは，Linuxのブリッジ機能を使うネットワークで，作成されたコンテナがデフォルトで接続されるネットワークである．

Bridgeネットワークでは，同じネットワーク内に存在するコンテナとIPアドレス指定で通信することができる．ただし，Bridgeネットワークでは、名前解決が定義されていない為，コンテナ名指定で通信することはできない．また、デフォルト状態では外部に公開されていないネットワークだが，-p オプションで指定したポートを解放することで，外部からコンテナにアクセスできるようになる．

Hostネットワーク

Hostネットワークは，コンテナがホスト側のネットワークインターフェースを共有するネットワークである．ネットワークを隔離しない為，コンテナ内でポートをリッスンすると，そのままインターネット側と通信可能になる．例えば，Hostネットワークに接続したコンテナでNginxコンテナを実行した場合，Hostマシンの80番ポートでリッスンしている場合と，同等になる．
そのため，Bridgeネットワークのように，コンテナ起動時に-p オプションを設定して外部にポート公開せずとも，コンテナを起動しただけで，Hostマシンのポート番号に接続すれば，コンテナに接続することができる．

Noneネットワーク

Noneネットワークは，コンテナにネットワークインターフェースを持たせないネットワークであり，コンテナが内外の通信ができなくなる．Noneネットワークに接続する場合は，他のネットワークから全て切断させる必要がある．

独自ネットワーク

独自のネットワークを作成することで，コンテナ名でのコンテナ間通信が可能となる．
$ docker network create <ネットワーク名>で新しくネットワークを作成し，$ docker network connect <接続するネットワーク名> <コンテナ名>で，指定したコンテナを指定したネットワークに接続していく．

ユーザー定義の独自ネットワークでは，名前解決が有効化され，コンテナ名で他のコンテナに接続できるようになる．
なお，コンテナ起動時，どのネットワークに接続するかは「--net」オプションで指定する。
