Dockerとは

Dockerとは，Docker社が提供しているOSSのコンテナエンジンである（※有償版有）．現在，最も普及しているコンテナ型仮想化ソフトウェアであり，Amazon Web Service(AWS)のElastic Container Service(ECS)のようなコンテナプラットフォームや，開発者の開発環境など広く使われている．コンテナはプラットフォームに依存しない為，DockerはWindowsとLinuxベースの両方のプラットフォームで実行できる．実際，Do仮想マシン内でDockerを実行することも可能である．Dockerの主な目的は，分散アーキテクチャでマイクロサービス・アプリケーションを実行できるようにすることである．

Dockerは以下のような機能・特徴を持つ

  • 可搬性・共有機能
    Dockerは，コンテナをtarファイルにアーカイブして他の物理マシンに送信したり，コンテナレジストリを介して共有できる機能を持つ．これによって，コンテナの他の環境に流用することができる．
  • IaC（Infrastructure as Code）
    IaCとは，コードやスクリプトを用いてインフラストラクチャの管理やプロビジョニングを行うことである．
    Dockerでは，コンテナ内に作成するファイルやインストールするアプリケーションなどを，Dockerfileという形式で定義できる．これによって，IaC的にコンテナの内部を定義することができる．
  • バージョン管理
    Dockerで作成したコンテナはバージョン番号を付与でき．Dockerで管理することができる．これによって，差分管理やロールバックなどが実現できる．
  • エコシステム
    Dockerは，取り巻くエコシステムも多い．例として，以下のような関連ソフトウェアやサービスがある．
    • Docker Hub ： Dockerイメージの管理や共有を行うサイト（パブリックリポジトリ）
    • Docker-Compose ： 複数のコンテナを連携して立ち上げるソフト
    • Kubernetes ： コンテナの運用管理と自動化を行うために設計されたOSS

Docker Architecture

Dockerは．クライアント・サーバ・アーキテクチャを採用している．DockerクライアントはDockerデーモンと通信し，Dockerコンテナの構築・実行・配布といった重要な作業を行う．基本的に，Dockerクライアントとデーモンは同じシステム上で動作させることができ，また、DockerクライアントをリモートのDockerデーモンに接続することも可能である．Dockerクライアントとデーモンは，REST APIを使用して，UNIXソケットまたはネットワークインタフェース上で通信する．また，DockerクライアントにはDocker Compose（※後述）がありコンテナ群からなるアプリケーションを操作することができる．

Dockerクライアント

Dockerのユーザは，クライアントを介してDockerと対話することができる．$ docker runのようなコマンドを実行すると，クライアントはそれをdockerデーモンに送信し，dockerデーモンがそれを実行する．Dockerコマンドは，Docker APIを使用する．なお，Dockerクライアントは、複数のデーモンと通信することができる．

Dockerホスト

Dockerホストは，アプリケーションの実行および実行するための環境を提供する．Dockerデーモン，イメージ，コンテナ，ネットワーク，ストレージから構成される．

Dockerデーモン

Dockerデーモン（dockerd）は，コンテナに関連するすべてのアクションを担当し，CLIまたはREST APIを介してコマンドを受け取る．また，他のデーモンと通信し，Dockerのサービスを管理することもできる．

Dockerレジストリ

Dockerレジストリは，イメージを保存したり，ダウンロードしたりできる場所を提供するサービスである．パブリックなリポジトリとしては，Docker Hubが提供されており，DockerはデフォルトでDocker Hub上のイメージを検索するように設定されている．また，自分専用のプライベートなレジストリを運用することもできる．$ docker pullや$ docker runを実行すると，設定したレジストリから必要なイメージが引き出される．また，$ docker pushを実行すると，設定したレジストリにイメージがプッシュされる．
